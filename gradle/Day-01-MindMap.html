⚡ Groovy DSL vs Kotlin DSL — Head to Head

GROOVY DSL

🟡 build.gradle

  • Dynamic typing — flexible
  • Single quotes for strings
  • No parentheses needed
  • Shorter, quicker to write
  • Older projects use this
  • Less IDE auto-complete
dependencies {
    implementation 'org.springframework
        .boot:spring-boot-starter:3.1.0'
    testImplementation 'junit:junit:4.13'
}
VS
KOTLIN DSL

🔵 build.gradle.kts

  • Static typing — strict, safe
  • Double quotes for strings
  • Parentheses are required
  • More explicit/verbose
  • Modern/new projects use this
  • Full IDE auto-complete ✅
dependencies {
    implementation("org.springframework
        .boot:spring-boot-starter:3.1.0")
    testImplementation("junit:junit:4.13")
}

WRAPPER

🎁 Gradle Wrapper Explained

Run Gradle without installing it on your machine.

gradlew.bat build   ← Windows
./gradlew build     ← Linux/Mac

Files:

  • gradlew — Linux/Mac script
  • gradlew.bat — Windows script
  • gradle-wrapper.jar — downloads Gradle
  • gradle-wrapper.properties — version info
STRUCTURE

📁 Project Structure

my-project/
├── build.gradle        ← build config
├── settings.gradle     ← project name
├── gradlew.bat         ← wrapper (Windows)
├── gradle/wrapper/     ← version files
├── src/
│   ├── main/java/      ← your code
│   ├── main/resources/ ← configs
│   ├── test/java/      ← tests
│   └── test/resources/
└── build/              ← output
